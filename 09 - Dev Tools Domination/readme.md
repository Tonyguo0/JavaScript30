# Learnt

- Dev tools are a powerful tool for debugging and optimizing web applications. They allow you to inspect the DOM, view network requests, and analyze performance.
- The Elements panel allows you to inspect the DOM and CSS of a web page. You can view the HTML structure, modify styles, and see how changes affect the page in real-time.
- `console.log()` is a useful method for debugging JavaScript code. It allows you to output messages to the console, which can help you understand the flow of your code and identify issues.
- you can style `console.log()` output using CSS. By passing a string with CSS styles as the second argument to `console.log()`, you can customize the appearance of your log messages.
- can use `console.assert()` to log messages only when a certain condition is false. This can help you identify issues without cluttering your console with unnecessary logs.
- can use `console.group()` and `console.groupEnd()` to organize related log messages into collapsible groups. This can make it easier to navigate through your console output.
- can use `console.time()` and `console.timeEnd()` to measure the time taken by a block of code. This can help you identify performance bottlenecks in your application.
- can use `console.dir()` to display an interactive list of the properties of a specified JavaScript object. This can be useful for inspecting complex objects and understanding their structure.
- can use `console.count()` to count the number of times a particular log message has been printed. This can help you track how many times a certain code path is executed.
- can use `console.table()` to display tabular data in a more readable format. This can be especially useful for arrays of objects, allowing you to easily compare properties across multiple items.
- `console.warn()` and `console.error()` and `console.info()` can be used to log messages with different levels of severity. This can help you differentiate between warnings, errors, and informational messages in your console output.
