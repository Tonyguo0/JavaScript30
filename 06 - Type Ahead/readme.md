# Learnt

- `type=module` in script tag to use ES6 module syntax in browser which allows us to use async/await in the top level of our JavaScript code without needing to wrap it in an async function. This can make our code cleaner and easier to read, as we can use async/await directly in our main script without needing to create additional functions or callbacks. this works because when we use `type=module`, the JavaScript code is treated as a module and is executed in strict mode, which allows us to use modern JavaScript features like async/await without any issues. This can help improve the readability and maintainability of our code, as we can write asynchronous code in a more straightforward and intuitive way.
- however we need to make sure that our JavaScript file is served from a web server, as some browsers may not allow us to use ES6 modules when opening the HTML file directly from the file system. This is because of security restrictions that prevent certain features from being used in local files. To avoid this issue, we can use a simple web server like `http-server` or `live-server` to serve our files and enable the use of ES6 modules in the browser. This can help ensure that our code runs smoothly and without any issues related to module loading or security restrictions.
- also variables are scoped to the module when using `type=module`, which means that they are not accessible from the global scope. This can help prevent naming conflicts and improve the overall organization of our code, as we can keep our variables and functions contained within the module without worrying about them interfering with other parts of our code or third-party libraries.
